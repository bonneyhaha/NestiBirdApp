 3. APPLICATION FLOW
────────────────────────────────────────────────────────────────────────────────

 STARTUP
 ───────
 1. App.xaml launches MainWindow.
 2. Window_Loaded fires:
      a. Window positioned at bottom-right of WorkArea (–5 px padding).
      b. WM_NCHITTEST hook installed — selective click-through begins.
      c. UserHelper.GetSystemUsername() reads Windows login name.
      d. UserHelper.GetFullNameAsync() calls REST API (if configured).
      e. Greeting bubble text set: "Good Morning, Jane!"
      f. Reads USE_REAL_WEBSOCKET from .env:
           true  → WebSocketService:
                     ResolveUrlAsync() → GET API or WS_URL fallback
                     ConnectAsync(wsUrl) starts the receive loop.
           false → DummyWebSocketService:
                     ConnectAsync() starts DispatcherTimer.
                     Fires rotating fake notifications every DUMMY_INTERVAL_MS.

 CLICK-THROUGH (WM_NCHITTEST)
 ─────────────────────────────
 Windows sends WM_NCHITTEST before every mouse event.
 Our WndProc hook intercepts it and:
   • Returns HTTRANSPARENT (−1) for empty transparent areas
     → clicks fall through to the desktop / apps underneath.
   • Returns HTCLIENT (1) for:
       – BirdContainer (including the 32 px button row)
       – GreetingBubble when Visible (it extends left outside BirdContainer)
       – Any active notification card in NotificationsPanel
     → window receives the click normally.
 No WS_EX_TRANSPARENT flag toggling needed — cleaner, no flicker.

 NOTIFICATION RECEIVED
 ──────────────────────
 WebSocketService / DummyWebSocketService (background / timer thread)
   → Fires NotificationReceived event with NotificationMessage

 MainWindow.OnNotificationReceived (marshalled to UI thread via Dispatcher.Invoke)
   → Checks DedupeKey — drops duplicate messages
   → If bird was hidden: restores BirdContainer + NotificationsPanel
   → If at MAX_NOTIFICATIONS: removes oldest card (no animation)
   → Creates NotificationControl, calls SetData(id, title, body, url)
   → Adds card to NotificationsPanel (StackPanel, bottom-right stacked)
   → NotificationControl.Loaded fires → slide-in animation plays
   → Starts DispatcherTimer (NOTIFICATION_DURATION_MS)
   → SoundService.Play() plays bird_chirp.mp3

 NOTIFICATION AUTO-DISMISS
 ──────────────────────────
 DispatcherTimer.Tick fires after NOTIFICATION_DURATION_MS:
   → RemoveNotification(ctrl, animate:true)
   → NotificationControl.AnimateOut() slides card out to the right
   → Callback removes control from StackPanel

 USER SNOOZES (snooze button on card)
 ──────────────────────────────────────
 Snooze_Click fires:
   → NotificationApiService.SnoozeAsync(notificationId)
       POST MARS_SNOOZE_URL  { "id": "…" }  (fire-and-forget, no crash on failure)
   → SnoozeRequested event raised
   → MainWindow catches it → RemoveNotification(ctrl) → AnimateOut()

 USER DISMISSES (× button on card)
 ────────────────────────────────────
 Dismiss_Click fires:
   → NotificationApiService.MarkAsReadAsync(notificationId)
       POST MARK_AS_READ_URL  { "id": "…" }
       + stores id to %AppData%\Nesti\dismissed.json
   → DismissRequested event raised
   → MainWindow catches it → RemoveNotification(ctrl) → AnimateOut()

 CARD CLICKED (body area)
 ─────────────────────────
 Card_Click → Process.Start(notificationUrl) → opens in default browser.

 BIRD HOVER
 ──────────
 MouseEnter on BirdContainer:
   → ControlButtons (Hide / Close PNG icons) → Visible
   → GreetingBubble fades in (opacity 0 → 1, 200 ms)

 MouseLeave on BirdContainer:
   → ControlButtons → Collapsed
   → GreetingBubble fades out (opacity 1 → 0, 200 ms) → Collapsed

 Note: BirdContainer uses a 2-row Grid (Row 0 = 32 px buttons, Row 1 = bird GIF).
 Buttons are INSIDE the layout bounds so hover is detected correctly.
 GreetingBubble is also checked explicitly in IsInteractive() because it
 extends outside the BirdContainer bounds to the left.

 HIDE BUTTON
 ───────────
 _birdHidden = true
 BirdContainer and NotificationsPanel → Collapsed
 WndProc returns HTTRANSPARENT for the entire window (fully click-through).
 On next notification received → bird and panel are automatically restored.

 CLOSE BUTTON
 ────────────
 Application.Current.Shutdown() → Window_Closing → WebSocketService.Dispose()

 RECONNECTION (real WebSocket only)
 ────────────────────────────────────
 Connection drop detected in ListenAsync loop
 → ScheduleReconnectAsync():
     attempt 1 : wait  2 s  (WS_RECONNECT_BASE_MS × 2^0)
     attempt 2 : wait  4 s
     attempt 3 : wait  8 s
     …
     attempt N : wait min(base × 2^(N−1), 60 000) ms
     > MAX_ATTEMPTS : StatusChanged("failed"), stop retrying.

 4. BUILD & RUN COMMANDS
────────────────────────────────────────────────────────────────────────────────

 Run all commands from the PROJECT folder:
   cd "POC-2-WPF\Nesti\Nesti"

 ── DEVELOPMENT ────────────────────────────────────────────────────────────────

 Restore NuGet packages (first time, or after adding a package):
   dotnet restore

 Build (Debug):
   dotnet build --configuration Debug

 Run (Debug, prints WebSocket status to debug output):
   dotnet run

 Run the compiled exe directly (no console window):
   bin\Debug\net8.0-windows\Nesti.exe

 Quick test without a real server — set in .env:
   USE_REAL_WEBSOCKET=false
   DUMMY_INTERVAL_MS=3000

 ── PRODUCTION ─────────────────────────────────────────────────────────────────

 Build (Release, optimised):
   dotnet build --configuration Release

 Publish — framework-dependent (requires .NET 8 on target machine):
   dotnet publish -c Release -r win-x64

   Output: bin\Release\net8.0-windows\win-x64\publish\

 Publish — self-contained single EXE (no .NET required on target machine):
   dotnet publish -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true

   Output: bin\Release\net8.0-windows\win-x64\publish\Nesti.exe  (~60 MB)

 IMPORTANT — files that must travel WITH Nesti.exe:
   .env                    ← configuration (edit to change URLs, no recompile)
   assets\bird_chirp.mp3   ← notification sound


────────────────────────────────────────────────────────────────────────────────
 5. EDITING CONFIGURATION WITHOUT REBUILDING
────────────────────────────────────────────────────────────────────────────────

 During development:
   Edit  Nesti\Nesti\.env
   The build copies it to the output folder automatically.

 On a deployed machine:
   Edit  <install-dir>\.env  (the copy next to the exe)
   Restart Nesti.exe — changes take effect immediately.

 You do NOT need to recompile to change any URL, timeout, or feature flag.

 8. LOCAL DISMISSED IDs STORE
────────────────────────────────────────────────────────────────────────────────

 Path:   %AppData%\Nesti\dismissed.json

 Format: ["id-1", "id-2", …]

 Populated by: NotificationApiService.MarkAsReadAsync()
 Checked by:   NotificationApiService.IsDismissed()

 Purpose: Prevents previously dismissed notifications from reappearing
          after an app restart (call IsDismissed before showing any card).

 9. RAM & GLASSMORPHISM NOTES
────────────────────────────────────────────────────────────────────────────────

 Element                      Cost       Notes
 ───────────────────────────  ─────────  ─────────────────────────────────────
 DropShadowEffect             HIGH       Creates an offscreen GPU bitmap per
                                         element. BlurRadius has been reduced:
                                         Notification card: 20 → 12
                                         GreetingBubble:    16 → 8
                                         Remove entirely to free the most RAM.

 Animated GIFs                MEDIUM     All frames decoded and held in memory.
                                         nest_bird.gif (~150 frames) is the
                                         largest. Replace with static PNG to
                                         eliminate this cost.

 Semi-transparent Background  LOW        WPF compositing — no extra bitmaps.
 (#CCFFFFFF, #40FFFFFF)                  Safe to keep.

 CornerRadius on Border        LOW        Clip mask only. Negligible.

 LinearGradientBrush           LOW        GPU gradient, no bitmap. Negligible.

 To reduce RAM further (in priority order):
   1. Remove DropShadowEffect from NotificationControl.xaml and/or
      GreetingBubble in MainWindow.xaml.
   2. Replace animated jarvis.gif with a static PNG icon.
   3. Replace nest_bird.gif with a static PNG bird.
